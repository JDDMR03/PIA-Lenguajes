Compilador

Un compilador es un software que traduce el código fuente de un programa a otro tipo de lenguaje para que pueda ser interpretado por la máquina. El primer compilador conocido fue creado por la doctora en matemáticas y militar estadounidense Grace Hopper en 1949. Un compilador consta de dos partes principales:

    Front End: Realiza un análisis al código fuente que se utilizará.

    Back End: Construye el código en un formato entendible para el PC después de que el front end analiza el código fuente.

Los compiladores se pueden clasificar en varios tipos según diferentes criterios:

    Según el número de pasadas: Los compiladores de una sola pasada leen el código fuente una vez y generan el código objeto. Los compiladores de múltiples pasadas leen el código fuente varias veces antes de generar el código objeto.

    Según el lenguaje de salida: Los compiladores nativos generan un código objeto específico para la plataforma en la que se ejecutan. Los compiladores cruzados generan un código objeto para una plataforma diferente a la que se están ejecutando.

    Según el momento de compilación: Los compiladores estáticos compilan el código fuente antes de la ejecución. Los compiladores dinámicos o JIT (Just-In-Time) compilan el código durante la ejecución.

Intérprete

Un intérprete es un programa informático que procesa el código fuente de un proyecto de software durante su tiempo de ejecución, es decir, mientras el software se está ejecutando, y actúa como una interfaz entre ese proyecto y el procesador. Un intérprete siempre procesa el código línea por línea, de modo que lee, analiza y prepara cada secuencia de forma consecutiva para el procesador.

Los intérpretes también se pueden clasificar en varios tipos:

    Intérpretes de línea de comandos: Estos intérpretes permiten a los usuarios introducir y ejecutar comandos una línea a la vez.

    Intérpretes de scripts: Estos intérpretes leen y ejecutan archivos de script, que son programas almacenados en un archivo.

    Intérpretes Just-In-Time (JIT): Estos intérpretes compilan el código fuente a un formato intermedio (bytecode) y luego lo interpretan durante la ejecución.
